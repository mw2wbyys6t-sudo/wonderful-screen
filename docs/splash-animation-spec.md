# 星云编年史 · 启动动画需求规格书

> 版本：v1.0  
> 日期：2026-06-24  
> 状态：待审查  
> 用途：初赛作品启动页 / 开机动画

---

## 1. 项目背景

《星云编年史》目前拥有 8 张历届世萌萌王角色海报，已用于登录页背景轮播。为了提升作品初亮相时的仪式感与视觉冲击力，用户希望在进入登录页/主应用之前，先播放一段「琉璃玻璃球包裹角色」的启动动画，最终浮现作品名，再进入正式页面。

---

## 2. 需求目标

- 制作一段 8~10 秒的启动动画（Splash / Boot Animation）。
- 8 位角色分别被封装在「动态琉璃玻璃球」中。
- 玻璃球会整体旋转、分散、聚合，类似高端手机或游戏的开机动画。
- 动画结尾浮现作品名《星云编年史 NEBULA CHRONICLE》。
- 动画结束后进入登录页 `/login.html`（或主应用 `/nebula-chronicle.html`，可配置）。
- 保持初赛阶段改动量可控，不重构整个项目架构。

---

## 3. 动画分镜脚本（时间轴）

| 阶段 | 时间 | 画面描述 | 音效预留 |
|---|---|---|---|
| **Phase 0: 暗场** | 0.0s ~ 0.5s | 纯黑背景，中央出现微弱光点 | 无 / 低频嗡鸣起势 |
| **Phase 1: 光球诞生** | 0.5s ~ 1.5s | 光点膨胀为一个半透明的琉璃玻璃球，内部隐约显现第一位角色（如薇尔莉特） | 玻璃凝结音效 |
| **Phase 2: 分裂扩散** | 1.5s ~ 3.5s | 玻璃球分裂为 8 个较小的玻璃球，向四周散开，排列成三维圆环/球面阵列；每个球自转并绕中心公转 | 清脆的分裂+扩散音 |
| **Phase 3: 环绕展示** | 3.5s ~ 5.5s | 8 个玻璃球持续缓慢公转，球体自身也缓慢自转；角色图像在球内清晰可见，每个球边缘有对应主题色微光 | 轻柔环境音 |
| **Phase 4: 聚合融合** | 5.5s ~ 6.5s | 8 个玻璃球向中心聚合，逐渐融合成一个发光的巨大琉璃球 | 聚合/能量汇聚音效 |
| **Phase 5: 爆裂显字** | 6.5s ~ 7.5s | 大玻璃球爆散成光粒，光粒中浮现「星云编年史」中文标题与「NEBULA CHRONICLE」英文副标题 | 爆裂+标题显现音效 |
| **Phase 6: 入口就绪** | 7.5s ~ 10.0s | 标题稳定显示，下方出现「进入」按钮或提示语；用户点击/按键后进入登录页 | 结束提示音 |
| **Phase 7: 过渡离场** | 10.0s+ | 标题淡出，登录页淡入；或直接进入登录页 | 无 |

## 4. 视觉设计规范

### 4.1 玻璃球外观

- **材质**：半透明琉璃/液态玻璃（Liquid Glass）。
- **形状**：默认球体（Sphere），可备选圆角立方体（Squircle Cube）。
- **光学效果**：
  - 菲涅尔高光边缘（Fresnel rim light）
  - 内部轻微折射/扭曲
  - 玻璃表面动态高光随旋转移动
  - 每个玻璃球边缘带角色主题色辉光（如夏娜红、雷姆蓝）
- **内部内容**：角色海报以贴图形式映射在球体内部或悬浮于球心。

### 4.2 动效风格

- **整体运动**：平滑、带弹性缓冲（ease-out / spring），避免生硬线性。
- **旋转**：
  - 公转：8 球围绕中心 Y 轴缓慢旋转（约 8~12 秒/圈）。
  - 自转：每个球体自身缓慢旋转（约 12~16 秒/圈）。
- **聚散动画**：
  - 扩散时：从中心向外呈爆炸式弧线运动，带轻微延迟错落（stagger）。
  - 聚合时：从四周向中心加速回收，最后轻微过冲再稳定。

### 4.3 色彩与氛围

- **背景**：深空黑 `#03030a` 到 `#0a0a18`，带微弱星云噪点。
- **玻璃高光**：冷白 `#ffffff` + 角色主题色。
- **标题文字**：
  - 中文「星云编年史」：大字、无衬线、白色、轻微发光。
  - 英文「NEBULA CHRONICLE」：小字、等宽/科技感字体、霓虹青蓝 `#00f3ff`。

### 4.4 字体

- 中文标题：Noto Sans SC / 思源黑体 Bold
- 英文副标题：Inter / Poppins / Space Grotesk

## 5. 技术实现方案

### 5.1 方案 A：Three.js WebGL（推荐）

- 使用 Three.js 创建 3D 场景。
- 每个玻璃球使用 `MeshPhysicalMaterial` 模拟透光/折射/清漆高光。
- 球体贴图：8 张角色海报。
- 使用 `PointLight` + `AmbientLight` 营造玻璃质感。
- 动画使用 GSAP 或 Tween.js 控制位置、旋转、缩放。
- 最后阶段用 CSS 覆盖层显示标题。

**优点**：效果最炫，3D 感真实，支持玻璃折射。  
**缺点**：需要加载 Three.js，低端设备可能帧率下降。

### 5.2 方案 B：纯 CSS 3D（轻量备选）

- 使用 CSS `transform-style: preserve-3d` + `rotateY/rotateX` + `translateZ`。
- 玻璃球用多层 radial-gradient + blur + border 模拟。
- 图片用 `border-radius: 50%` 或球面遮罩。
- 动画用 CSS `@keyframes` 控制公转、自转、聚散。

**优点**：轻量、无需 WebGL、加载快。  
**缺点**：玻璃质感较弱，3D 纵深有限。

### 5.3 最终推荐

- **初赛阶段采用方案 A（Three.js）**，因为用户明确要求「琉璃玻璃」和「转圈分开」的 3D 效果，纯 CSS 难以达到同等质感。
- 提供「跳过动画」按钮与 `prefers-reduced-motion` 媒体查询降级。

## 8. 交互逻辑

1. 用户打开 `splash.html`。
2. 页面预加载 Three.js 与角色贴图。
3. 加载完成后自动播放动画。
4. 动画全程可点击「跳过」直接进入登录页。
5. Phase 6 显示标题与「进入」按钮，等待用户交互。
6. 用户点击/按空格/回车后，淡出并跳转至 `/login.html`。
7. 增加 `localStorage` 标记：`hasSeenSplash=true`，下次访问可选择直接跳过（可选）。

---

## 9. 性能与兼容性

- **目标帧率**：桌面 60fps，移动端 30fps+。
- **降级策略**：
  - WebGL 不支持时，降级为静态标题页或 CSS 简化版。
  - 低性能设备减少玻璃球数量、关闭折射、降低分辨率。
- **无障碍**：
  - 支持 `prefers-reduced-motion` 关闭动画。
  - 跳过按钮始终可聚焦。

## 10. 待确认事项

请在审查后确认或修改以下内容：

1. **动画时长**：建议 8~10 秒，是否需要缩短或延长？
2. **玻璃球形状**：优先球体，是否接受圆角立方体？
3. **角色排列**：扩散后是排成圆环（2D 平面）还是球面（3D 空间）？
4. **结束去向**：动画结束后进入 `/login.html` 还是 `/nebula-chronicle.html`？
5. **是否自动跳转**：Phase 6 后是否等待用户点击，还是 2 秒后自动进入？
6. **跳过机制**：是否支持「首次播放后不再显示」？
7. **音效**：初赛是否先不加音效，仅做视觉？
8. **技术方案**：是否同意使用 Three.js 实现？
