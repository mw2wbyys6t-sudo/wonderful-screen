# 星云编年史 · 启动页/登录页一体化设计规格书

> 版本：v2.0  
> 日期：2026-06-24  
> 状态：待审查  
> 用途：初赛作品启动页 + 登录页（一体化沉浸式体验）

---

## 1. 项目背景

《星云编年史》已拥有 8 张历届世萌萌王角色海报。当前登录页采用背景轮播展示角色，视觉层级较平。为了在比赛初赛中建立强烈的第一印象，计划将 `/workspace/liquid-glass-demo.html` 升级为「启动动画 + 登录界面」一体化页面，突出：

- **星云世界观**
- **8 位角色构成的星图**
- **Apple Liquid Glass 质感**
- **无缝衔接的登录入口**

---

## 2. 设计目标

让用户打开页面后立刻感受到：

> 「这个项目很高级」  
> 「这不像普通网页」  
> 「这有 Apple 发布会级别的质感」

核心叙事：

```
星云苏醒
  ↓
液态玻璃凝聚
  ↓
角色觉醒
  ↓
编年史展开（星座/星图）
  ↓
星图坍缩 → 星云漩涡
  ↓
标题浮现
  ↓
登录界面升起
```

---

## 3. 核心设计理念

### 3.1 不要做

- 简单的人物卡片转圈
- 卡牌聚合后跳转到登录页（世界断裂）
- 过度卡通化的魔法少女风格

### 3.2 要做

- **液态玻璃是表现手法**，不是主体
- **8 位角色是主角**，以星图/星座方式呈现
- **星云编年史的世界观是核心**，从开场到登录结束保持一致
- **单页沉浸**，启动动画和登录界面在同一页面无缝过渡

### 3.3 视觉参考融合

- Apple Liquid Glass
- Apple Vision Pro UI
- Honkai: Star Rail 启动画面
- Fate/Grand Order 周年 PV
- NieR: Automata UI

关键词：**透明、流动、神秘、未来、高贵、史诗感**

## 4. 动画分镜脚本（时间轴）

总时长约 12~15 秒，之后自然过渡到登录界面。

| 阶段 | 时间 | 画面描述 | 镜头/运动 |
|---|---|---|---|
| **Phase 1: 星云苏醒** | 0.0s ~ 2.0s | 屏幕几乎全黑，深空星点缓慢漂浮，蓝紫色光雾弥漫。中央浮现一颗液态玻璃水滴，像水银一样缓慢漂浮、呼吸 | 缓慢推进，聚焦中央 |
| **Phase 2: 液态玻璃凝聚** | 2.0s ~ 3.5s | 水滴分裂/延展为 8 个玻璃碎片/薄片，每个碎片内部开始有微光，角色图像从模糊到清晰显现 | 轻微拉远，展示碎片分布 |
| **Phase 3: 角色觉醒** | 3.5s ~ 5.5s | 8 个玻璃碎片定型为竖版液态玻璃角色卡片；角色从半透明到完全清晰；每张卡片有独立主题色辉光 | 环绕缓慢旋转 |
| **Phase 4: 编年史展开** | 5.5s ~ 8.0s | 8 张角色卡片围绕中心排列成星图/星座（菱形/环形），卡片之间出现发光连线；中央浮现小型星云/魔法阵 | 缓慢公转 + 自身微倾 |
| **Phase 5: 星图坍缩** | 8.0s ~ 9.5s | 发光连线收束，角色卡片开始向中心缓慢移动；卡片边缘液化，拖出粒子尾迹 | 向中心缓慢推近 |
| **Phase 6: 星云漩涡** | 9.5s ~ 11.0s | 角色卡片完全液化/粒子化，旋转汇聚成巨大的星云漩涡；漩涡中央极度明亮 | 俯冲进入漩涡中心 |
| **Phase 7: 标题浮现** | 11.0s ~ 13.0s | 漩涡绽放，光粒四散，中央浮现「星云编年史」中文标题与「NEBULA CHRONICLE」英文副标题；标题背后有悬浮玻璃圆盘 | 静止，突出标题 |
| **Phase 8: 登录界面升起** | 13.0s+ | 标题和背景保留；角色卡片缩小并退到画面两侧（左侧 4 张，右侧 4 张）；中央升起 Liquid Glass 登录卡片 | 固定机位，UI 滑入 |

## 5. 视觉设计规范

### 5.1 色彩体系

- **背景深空**：`#050714` → `#03030a`
- **星云光雾**：蓝紫渐变 `#4da6ff` / `#7c4dff` / `#b892ff`，透明度 15%~35%
- **星点**：纯白 `#ffffff`，大小随机，带闪烁
- **玻璃高光**：冷白 `#ffffff`，动态流动
- **标题**：中文纯白微发光，英文霓虹青蓝 `#00f3ff`
- **角色主题色辉光**：根据角色提取（夏娜红、雷姆蓝、御坂电光蓝等）

### 5.2 液态玻璃卡片结构（5 层）

```
第 5 层：体积辉光（box-shadow，产生悬浮感）
第 4 层：高光边缘（glass rim，像玻璃切面）
第 3 层：动态光流（缓慢移动的 linear-gradient）
第 2 层：磨砂玻璃（backdrop-filter: blur(30px)）
第 1 层：角色图像
```

### 5.3 字体

- 中文标题：Noto Sans SC / 思源黑体 Bold
- 英文副标题：Space Grotesk / Inter
- 登录表单：系统默认无衬线

### 5.4 粒子与特效

- 星点：大量小点，缓慢闪烁、漂移
- 光雾：3~5 个大尺度模糊光斑，缓慢漂浮
- 星云连线：卡片之间的贝塞尔曲线，发光脉冲
- 漩涡粒子：角色液化时释放的金色/白色粒子
- 玻璃反光：卡片表面随视角变化的动态高光

## 7. 技术实现方案

### 7.1 核心技术栈

- **Three.js**：3D 玻璃卡片、粒子、漩涡、连线
- **GSAP**：动画时间线编排
- **HTML/CSS**：登录表单、标题、背景光斑
- **Canvas 2D**：星点背景
- **Modellix API（ChatGPT）**：设计文案、提示词优化
- **Modellix API（Imagen 4）**：如需生成新的背景/角色素材

### 7.2 关键实现细节

#### 玻璃卡片

- 使用 `THREE.MeshPhysicalMaterial`
- 参数：`transmission: 0.9, roughness: 0.15, thickness: 0.5, clearcoat: 1.0`
- 贴图：8 张角色海报
- 形状：竖版 Plane，圆角通过 alphaMap 或 Shader 实现

#### 星座连线

- 计算 8 张卡片在世界坐标中的位置
- 使用 `THREE.Line` 或 `THREE.BufferGeometry` 绘制发光线段
- 材质使用 `THREE.LineBasicMaterial` + additive blending

#### 星云漩涡

- 粒子系统 `THREE.Points`
- 粒子沿螺旋轨道向中心运动
- 颜色从角色主题色渐变到白色

#### 登录框升起

- CSS `transform: translateY()` + opacity
- 保留 WebGL 背景继续缓慢运行（角色在两侧轻微漂浮）

### 7.3 降级策略

- WebGL 不支持时：显示静态标题 + 登录卡片
- `prefers-reduced-motion`：关闭自动动画，直接显示登录界面
- 低性能设备：减少粒子数量、关闭折射、降低分辨率

## 9. 交互逻辑

1. 用户打开页面，自动播放启动动画。
2. 全程显示「跳过」按钮，点击直接进入登录界面。
3. Phase 7 标题出现后，显示「进入」提示。
4. Phase 8 登录界面自动升起，用户可输入账号密码。
5. 登录成功后跳转 `/nebula-chronicle.html`。
6. 支持键盘：空格/回车跳过动画或触发表单提交。
7. 可选 `localStorage` 标记首次访问，后续直接进入登录界面。
