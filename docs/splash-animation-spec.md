> 版本：v1.1  
> 日期：2026-06-24  
> 状态：待审查  
> 用途：初赛作品启动页 / 开机动画

---

## 1. 项目背景

《星云编年史》目前拥有 8 张历届世萌萌王角色海报，已用于登录页背景轮播。为了提升作品初亮相时的仪式感与视觉冲击力，用户希望在进入登录页/主应用之前，先播放一段启动动画：8 位角色以 **liquid-glass-demo.html 同款琉璃玻璃卡片** 的形式呈现，卡片会整体旋转、分散、聚合，类似高端手机或游戏的开机动画，最终浮现作品名，再进入正式页面。

---

## 2. 需求目标

- 制作一段 8~10 秒的启动动画（Splash / Boot Animation）。
- 8 位角色分别呈现在「动态琉璃玻璃卡片」中，视觉风格与 `/workspace/liquid-glass-demo.html` 的玻璃态系统保持一致。
- 玻璃卡片会整体旋转（轮播）、分散、聚合，类似高端手机或游戏的开机动画。
- 动画结尾浮现作品名《星云编年史 NEBULA CHRONICLE》。
- 动画结束后进入登录页 `/login.html`（或主应用 `/nebula-chronicle.html`，可配置）。
- 启动动画基于现有 `/workspace/liquid-glass-demo.html` 进行扩展，复用其深空背景、动态光斑、玻璃卡片质感、高光边框与鼠标跟随高光。
- 保持初赛阶段改动量可控，不重构整个项目架构。

## 3. 动画分镜脚本（时间轴）

| 阶段 | 时间 | 画面描述 | 音效预留 |
|---|---|---|---|
| **Phase 0: 暗场** | 0.0s ~ 0.5s | 深空黑背景，中央浮现微弱金色/白色魔法阵虚影 | 无 / 低频魔法嗡鸣 |
| **Phase 1: 卡牌诞生** | 0.5s ~ 1.5s | 魔法阵中央升起一张竖版琉璃卡牌，角色图像从模糊到清晰显现，卡牌边缘发光 | 卡牌凝结/出现音效 |
| **Phase 2: 分裂扩散** | 1.5s ~ 3.5s | 中央卡牌如施法般分裂为 8 张，呈扇形/螺旋向四周飞舞散开，排列成环形牌阵；每张卡牌自转并绕中心公转 | 清脆的分裂+飞舞音效 |
| **Phase 3: 环绕展示** | 3.5s ~ 5.5s | 8 张卡牌在环形牌阵中持续旋转、上下漂浮；中央魔法阵缓缓转动；卡牌边缘主题色辉光呼吸闪烁 | 轻柔魔法环境音 |
| **Phase 4: 聚合封印** | 5.5s ~ 6.5s | 8 张卡牌被魔力牵引，向中央螺旋收回，依次重叠叠合为一个发光的卡牌堆/巨大卡牌 | 聚合/封印音效 |
| **Phase 5: 爆裂显字** | 6.5s ~ 7.5s | 重叠的卡牌爆散成金色/白色光粒与符文，光粒中浮现「星云编年史」中文标题与「NEBULA CHRONICLE」英文副标题 | 解封+标题显现音效 |
| **Phase 6: 入口就绪** | 7.5s ~ 10.0s | 标题稳定显示，下方出现「进入」按钮或提示语；用户点击/按键后进入登录页 | 结束提示音 |
| **Phase 7: 过渡离场** | 10.0s+ | 标题淡出，登录页淡入；或直接进入登录页 | 无 |

## 4. 视觉设计规范

### 4.1 玻璃卡片外观（复用 liquid-glass-demo 风格）

- **材质**：半透明液态玻璃（Liquid Glass），与 `liquid-glass-demo.html` 的 `.glass-card` 一致。
- **形状**：竖版长方形卡牌（类似塔罗牌 / 库洛牌 / 魔法少女卡牌），宽高比约 2:3 或 3:4，圆角 16~24px，整体修长优雅。
- **光学效果**：
  - `backdrop-filter: blur(28px)` 磨砂玻璃质感
  - 顶部轮廓高光（白色渐变边框，mask-composite 实现）
  - 玻璃表面鼠标跟随高光（`.sheen`）
  - 卡片边缘带角色主题色辉光（box-shadow）
  - 多层深度：后排有装饰玻璃层 `.glass-back`
- **内部内容**：角色海报以图片形式置于玻璃卡片中央，图片本身可再覆盖一层微弱玻璃质感，使其与卡片融合。

### 4.2 动效风格（魔卡少女樱式召唤卡牌）

整体动画参考《魔卡少女樱》库洛牌召唤场景：卡牌从虚空中浮现、旋转飞舞、排列成阵、最终聚合释放魔法。

- **整体运动**：平滑、带魔法感，带弹性缓冲与轻微漂浮感。
- **旋转与飞舞**：
  - 公转：8 张卡牌围绕中心缓慢旋转，如牌阵展开（约 8~12 秒/圈）。
  - 自转：每张卡牌自身 3D 翻转（rotateY / rotateX），像被魔法托起在空中翻转。
  - 漂浮：卡牌有轻微的上下/左右漂浮，模拟被魔力托举。
- **聚散动画**：
  - 扩散时：从中央魔法阵向四周呈扇形/螺旋式飞散，带延迟错落（stagger）。
  - 聚合时：卡牌从四周被收回中央，如封印解除后归位，最后重叠成一个发光整体。
- **魔法特效元素**：
  - 中央若隐若现的魔法阵/符文环（金色或角色主题色）
  - 卡牌边缘发光描边，切换时闪烁
  - 粒子星光、漂浮光点、羽毛/花瓣（可选）
  - 卡牌飞过时有轻微拖尾残影

### 4.3 色彩与氛围

- **背景**：深空黑 `#050714`，复用 `liquid-glass-demo.html`。
- **动态光斑**：复用 `.orb` 蓝紫青三色光斑，缓慢漂浮。
- **玻璃高光**：冷白 `#ffffff` + 角色主题色。
- **标题文字**：
  - 中文「星云编年史」：大字、无衬线、白色、轻微发光。
  - 英文「NEBULA CHRONICLE」：小字、科技感字体、霓虹青蓝 `#00f3ff`。

### 4.4 字体

- 中文标题：Noto Sans SC / 思源黑体 Bold
- 英文副标题：Inter / Poppins / Space Grotesk

## 5. 技术实现方案

### 5.1 方案 A：Three.js WebGL 3D 卡片（推荐）

- 使用 Three.js 创建 3D 场景。
- 每张玻璃卡片用 `MeshPhysicalMaterial` 或 `ShaderMaterial` 模拟透光/清漆高光/折射。
- 卡片贴图：8 张角色海报。
- 使用 `PointLight` + `AmbientLight` 营造玻璃质感。
- 动画使用 GSAP 或 Tween.js 控制位置、旋转、缩放。
- 最后阶段用 CSS 覆盖层显示标题。
- **优点**：效果最炫，3D 纵深真实，玻璃折射自然。  
- **缺点**：需要加载 Three.js，低端设备可能帧率下降。

### 5.2 方案 B：纯 CSS 3D 卡片轮播（轻量备选）

- 使用 CSS `transform-style: preserve-3d` + `rotateY/rotateX` + `translateZ`。
- 玻璃卡片复用 `liquid-glass-demo.html` 的 `.glass-card` 样式。
- 图片用卡片内的 `img` 或 `background-image`。
- 动画用 CSS `@keyframes` 控制公转、自转、聚散。
- **优点**：轻量、无需 WebGL、加载快、与现有 glass-card 样式 100% 一致。  
- **缺点**：3D 纵深和玻璃折射弱于 Three.js。

### 5.3 最终推荐

- **初赛阶段采用方案 A（Three.js）**，因为用户明确要求「转圈」「分开」「聚合」的 3D 空间运动，Three.js 更容易做出卡片在三维空间中的弧线聚散。
- 同时保留 `liquid-glass-demo.html` 的玻璃态视觉语言（背景光斑、卡片高光、边框样式），让启动动画与该页面无缝融合。
- 提供「跳过动画」按钮与 `prefers-reduced-motion` 媒体查询降级。


## 8. 交互逻辑

1. 用户打开 `liquid-glass-demo.html`（或重命名后的启动页）。
2. 页面预加载 Three.js 与角色贴图。
3. 加载完成后自动播放动画。
4. 动画全程可点击「跳过」直接进入登录页/显示登录卡片。
5. Phase 6 显示标题与「进入」按钮，等待用户交互。
6. 用户点击/按空格/回车后，淡出并跳转至 `/login.html`（或本页登录卡片滑入）。
7. 增加 `localStorage` 标记：`hasSeenSplash=true`，下次访问可选择直接跳过（可选）。

---

## 9. 性能与兼容性

- **目标帧率**：桌面 60fps，移动端 30fps+。
- **降级策略**：
  - WebGL 不支持时，降级为静态标题页或 CSS 简化版。
  - 低性能设备减少卡片数量、关闭折射、降低分辨率。
- **无障碍**：
  - 支持 `prefers-reduced-motion` 关闭动画。
  - 跳过按钮始终可聚焦。
