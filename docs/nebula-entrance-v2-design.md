# 星云编年史 · 登录/入口页重设计 v2

> 基于 `/workspace/index.html`、`liquid-glass-demo.html`、`login-v2.html` 二次融合升级。
> 定位：**登录/入口页**，不是主界面。目标是在进入主应用前营造仪式感、二次元角色企划感与高端视觉冲击。
> 点击「开始旅程」后跳转至主应用（主应用不在本次设计范围，预留跳转接口）。

---

## 一、设计目标

1. **第一印象强烈**：登录页本身就是作品封面，像手游/动画启动画面。
2. **角色为核心**：8 位萌王角色以琉璃玻璃卡片形式出场，突出二次元属性。
3. **Apple 式克制排版**：标题与进入按钮干净、留白、高端。
4. **动态背景丰富**：加载时为华丽星云，展示阶段为角色视频轮播。
5. **为后续音效预留接口**：加载画面、角色动画、按钮点击均预置音效触发点。
6. **为社交传播优化**：页面本身要能录屏截图发出去获赞，具备"哇"的瞬间。

## 二、页面流程

```
[用户访问 index.html]
        │
        ▼
[Phase 0: 加载画面]
  • 华丽星云背景（粒子 + 光晕 + 暗角）
  • 中央转圈加载动画
  • 预加载：字体、图片、角色素材、2–3 个角色视频
  • 渲染技术：Three.js 可继续复用，也可根据效果需要替换/升级（如 WebGL 粒子、GSAP + CSS 3D、PixiJS 等）
  • 预留：背景氛围音效接口
        │
        ▼
[Phase 1: 琉璃玻璃角色展示]
  • 作品标题淡入
  • 8 位角色从星门中浮现，镶嵌在琉璃玻璃卡片中
  • 卡片环绕、旋转、散开重组（动画可稍显复杂，但整体时长不宜过长，节奏舒缓有仪式感）
  • 玻璃质感：折射、焦散、边缘光、动态高光
  • 整体效果：高雅且不失二次元吸引力，展开方式可参考《魔卡少女樱》库洛牌展开/封印解除的华丽感
        │
        ▼
[Phase 2: 登录/入口界面]
  • 角色动画结束，卡片自然淡出/收缩（过渡要足够华丽、平滑）
  • 背景切换为角色视频轮播（2–3 个短视频循环），过渡需平滑、具有商业价值感
  • Apple 风格「开始旅程」按钮浮现
  • 标题保持显示
        │
        ▼
[点击按钮]
  • 播放点击音效
  • 星门放大/白屏/粒子爆发过渡
  • 跳转至主应用（预留 `window.location.href` 接口，目标 URL 可配置）
```

## 三、视觉设计规范

### 3.1 色彩

| 用途 | 色值 | 说明 |
|------|------|------|
| 背景深空 | `#03030a` | 主背景，几乎纯黑 |
| 星云紫 | `#7c4dff` | 主光源、星门外层 |
| 琉璃青 | `#00f3ff` | 标题光晕、高亮边缘 |
| 角色粉 | `#ff2a6d` | 点缀色、角色卡片边缘 |
| 玻璃底色 | `rgba(255,255,255,0.08)` | 玻璃卡片填充 |
| 玻璃边框 | `rgba(255,255,255,0.18)` | 玻璃卡片描边 |
| 文字主色 | `#f5f5f7` | Apple 式白 |
| 文字次色 | `rgba(245,245,247,0.55)` | 副标题、说明 |

### 3.2 字体

- **标题**：`-apple-system, BlinkMacSystemFont, 'SF Pro Display', 'PingFang SC'`
  - 字重：300
  - 字号：clamp(24px, 5vw, 42px)
  - 字距：0.18em–0.22em
  - 全大写
- **副标题/按钮**：`SF Pro Text, PingFang SC`
  - 字重：400–500
  - 字距：0.1em–0.15em

### 3.3 玻璃卡片质感

- 背景：`backdrop-filter: blur(20px)` + 半透明白
- 边框：1px 半透明白 + 内发光
- 顶部高光：`inset 0 1px 0 rgba(255,255,255,0.2)`
- 动态光斑：鼠标移动时产生跟随高光
- 折射感：叠加轻微色差或棱镜边缘（可选 CSS `filter` 模拟）

## 四、动画分镜

### Phase 0: 加载画面（2.5–4 秒）

| 时间 | 内容 |
|------|------|
| 0.0s | 深空背景淡入，星云光晕开始脉动 |
| 0.3s | 中央加载环出现，细线旋转 |
| 0.5s | 背景粒子开始飘动 |
| 2.5s+ | 资源加载完成，加载环淡出 |

**升级点**：
- 背景从纯色改为多层 radial-gradient + 粒子
- 加载环改为双色渐变（青/紫）
- 增加「NEBULA CHRONICLE」水印式小字（opacity 0.15）

### Phase 1: 琉璃玻璃角色展示（6–8 秒）

| 时间 | 内容 |
|------|------|
| 0.0s | 标题从上方淡入 |
| 0.5s | 中央星门亮起，开始缓慢旋转 |
| 1.0s | 第一张角色玻璃卡从星门中心浮现 |
| 1.5s–4.0s | 其余 7 张卡片依次浮现，排列成环（stagger 依次出现） |
| 4.0s–6.0s | 环整体缓慢旋转，卡片轻微上下浮动 |
| 6.0s–7.0s | 卡片散开、放大、透明度降低，营造仪式感 |
| 7.0s–8.0s | 卡片华丽淡出，背景平滑过渡为视频轮播 |

**升级点**：
- 动画总时长控制在 6–8 秒，复杂但不拖沓
- 玻璃卡片增加折射、焦散、边缘光、动态高光
- 卡片浮现采用 stagger（依次出现）而非同时
- 散开与过渡阶段更华丽、自然，参考《魔卡少女樱》库洛牌展开的仪式感

### Phase 2: 入口界面（常驻）

| 元素 | 行为 |
|------|------|
| 背景视频 | 2–3 个短视频循环轮播，淡入淡出切换 |
| 标题 | 保持显示，轻微呼吸动画 |
| 进入按钮 | Apple 风格「开始旅程」从下方滑入，hover 时微亮/放大 |
| 点击 | 播放点击音效 + 按钮涟漪 + 星门放大/白屏/粒子爆发过渡 |

## 六、推荐 AI 工具 / 技能 / MCP

### 6.1 视频生成

| 工具 | 接入方式 | 优势 | 备注 |
|------|---------|------|------|
| **Seedance** | `byted-seedance-video-generate` skill | 字节跳动模型，本环境原生支持 | 优先尝试 |
| **MiniMax** | `mcp_minimax` 的 `generate_video` | 本环境已配置 MCP | 可作为备选 |
| **Kling V3** | 需通过 Modellix 或第三方 API | 可灵角色一致性好 | 需确认 API 可用性 |
| **Google Veo 3.1** | Modellix CLI | 已在本项目验证 | 可作为稳定备选 |

### 6.2 图像/首帧合成

| 工具 | 接入方式 | 用途 |
|------|---------|------|
| Python/PIL | 现有脚本改造 | 合成 8 角色首帧图 |
| Seedream | `byted-seedream-image-generate` skill | 如需要增强/修复角色图 |

### 6.3 音效（预配置阶段）

| 工具 | 接入方式 | 用途 |
|------|---------|------|
| MiniMax | `mcp_minimax` 的 `text_to_audio` / `music_generation` | 生成氛围音、点击音效 |

### 6.4 开发与测试

| 工具 | 接入方式 | 用途 |
