# 星云编年史 · 手势/语音观影功能设计文档

## 1. 背景与目标

在现有 [nebula-chronicle.html](file:///workspace/nebula-chronicle.html) 沉浸式手势交互系统基础上，新增「通过手势或语音观看动漫」的能力。用户选中动漫后，可跳转至独立播放页观看对应片源，后续可与 LLM 大模型对接（LLM 可直接生成 `watch.html?title=xxx` 链接）。

## 2. 设计原则

- **版权合规**：仅播放用户自行提供的合法片源（B 站免费视频或本地视频文件），系统不抓取、不存储、不破解任何版权内容。
- **懒人优先**：一句话语音即可直达播放；手势在无键盘鼠标场景下也能完成播放/暂停/退出。
- **渐进增强**：未配置片源时给出友好提示，不影响原有浏览体验。
- **独立可扩展**：播放页独立成页，方便 LLM 直接调用。

## 3. 数据模型变更

在 `rawAnimeData` 的每个对象中新增可选字段：

```js
{
  year: 1995,
  title: "新世纪福音战士",
  genre: "机甲",
  desc: "...",
  image: "images/14.jpg",
  bilibili: "BV1xx411c7mD",      // B 站 BV 号或完整 URL（可选）
  videoUrl: "videos/eva.mp4"     // 本地视频路径（可选）
}
```

`AnimeSchema` 的 `optional` 数组同步增加 `bilibili`、`videoUrl`。

### 片源优先级

1. 若 `bilibili` 存在且网络可用 → 使用 B 站 iframe 嵌入。
2. 否则若 `videoUrl` 存在 → 使用 `<video>` 播放本地文件。
3. 否则显示「暂无片源」占位，引导用户配置。

### B 站链接处理

- 支持 BV 号（如 `BV1xx411c7mD`）。
- 支持完整 B 站 URL，自动提取 BV 号。
- 嵌入地址统一为：`https://player.bilibili.com/player.html?bvid={bv}&page=1&high_quality=1&danmaku=0`。

## 4. 播放页 watch.html

新建独立页面 `/workspace/watch.html`：

### 页面结构

- 顶部：返回星云编年史按钮 + 当前动漫标题。
- 中部：视频播放区（B 站 iframe 或本地 `<video>`）。
- 底部：动漫简介、年份、类型标签。
- 角落：摄像头预览 + 手势提示 + 麦克风语音开关。
- 背景：复用 Three.js 星云背景，保持视觉一致性。

### 路由参数

- `watch.html?index=14`：按索引播放。
- `watch.html?title=新世纪福音战士`：按标题搜索播放（优先用于 LLM）。
- 无参数时显示「未指定动漫」提示页。

### 手势控制（播放页）

| 手势 | 动作 |
|------|------|
| 食指指向 | 移动光标悬停按钮 |
| 捏合（拇指+食指） | 点击：暂停/播放（点击播放/暂停按钮）；也可直接触发播放/暂停 |
| 双指滑动（食指+中指水平移动） | 快进/快退（仅本地视频有效，B 站 iframe 受跨域限制可能无效） |
| 五指张开 | 退出播放页，返回星云编年史 |

### 语音控制（播放页）

| 语音命令 | 动作 |
|----------|------|
| "播放" / "继续" | 播放视频 |
| "暂停" | 暂停视频 |
| "关闭" / "退出" / "返回" | 返回星云编年史 |
| "静音" / "取消静音" | 切换静音 |

## 6. 语音识别实现

- 使用浏览器原生 Web Speech API（`webkitSpeechRecognition` / `SpeechRecognition`）。
- 中文识别（`lang: 'zh-CN'`）。
- 持续监听（`continuous: true`），识别结果做防抖处理，避免重复触发。
- 无语音识别支持时自动隐藏麦克风按钮并降级为纯手势/鼠标。

## 7. 错误处理与降级

- **B 站视频无法播放**：自动尝试 `videoUrl` 本地视频；两者都失败时显示「该片源暂不可播放」。
- **摄像头未授权**：播放页仍可鼠标操作，语音功能不受影响（只要浏览器支持）。
- **语音识别未授权/不支持**：隐藏语音按钮，仅保留手势和鼠标。
- **跨页手势状态**：watch.html 独立初始化 MediaPipe，不依赖主页状态，避免跨页传递复杂对象。

## 8. 视觉风格

- 播放页背景复用星云主题，视频区采用深色玻璃面板。
- 控制按钮使用霓虹发光效果，与现有设计一致。
- 语音激活时麦克风图标显示脉冲动画。
- 返回按钮常驻左上角，便于随时退出。

## 10. 手势顺滑度优化（同步纳入本轮）

为让现有手势更跟手、更自然，本轮同步优化以下细节：

1. **光标平滑跟随**：手势光标位置采用 lerp 插值，减少抖动；静止时加入微小死区，避免光标漂移。
2. **手势防抖升级**：不同手势采用差异化防抖时长；pinch 增加「持续 150ms 才触发点击」的稳定期，减少误触。
3. **时间轴旋转阻尼**：手掌滑动旋转时，使用指数移动平均（EMA）平滑速度输入，避免忽快忽慢。
4. **悬停反馈柔化**：节点 hover/active 状态过渡更柔和，光标接近时增加外圈呼吸光环提示。
5. **五指张开放大判定窗口**：在主页检测 open 手势时加入小延迟，确认用户真的张开手掌后再打开详情，降低误触发。
6. **跨页手势一致性**：`watch.html` 复用同一套手势识别逻辑，光标阻尼、防抖参数与主页统一。
